# 常见查询类型模式

## 1. 趋势表 / table + 时间维度
特点：
- `vizData.chartType = table`
- `dimMetList` 往往带时间维度
- `vizData.datasets` 通常可展开成多行
- 常见输出：月 / 周 / 日趋势、分组趋势

优先关注：
- 时间维度真实粒度
- 展示标签范围与 SQL 时间边界是否一致
- 结果值是否需要按业务线/标签拆开

## 2. 指标卡片 / 单值总览
特点：
- 行数通常只有 1
- `fieldMap` 里多为 measures
- 常带环比 / 同比 / 原始值

优先关注：
- 当前值、对比值、变化率三者关系
- 对比窗口是自然周期还是“截至昨日的同窗口”

## 3. 总览统计 / 无维度聚合
特点：
- `dimMetList` 可能为空
- SQL 只有聚合指标，没有 GROUP BY
- 适合解读总体健康度

优先关注：
- 筛选范围
- 分母口径
- 指标是否基于参评量而非全量

## 4. 复杂派生口径查询
特点：
- 大量 `CASE WHEN`
- 指标依赖多个派生字段
- `expr` 与 `fullExpr` 差异明显

优先关注：
- 递归展开派生字段
- 不要把派生字段误当源表原始列
